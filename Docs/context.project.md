# Contexto de "Domus-Orbis" a nivel proyecto

## Nota

Esta documentación existe para preservar continuidad entre el problema observado, las decisiones tomadas y la solución que se construirá.

No está escrita como documentación pública ni como especificación exhaustiva. Está escrita principalmente para retomar el trabajo más adelante sin perder el contexto original.

## Contexto

El proyecto usa documentación colocalizada dentro de la solución para mantener cerca el entendimiento del dominio, las iteraciones y la implementación.

Esto permite que cada iteración preserve lo mínimo necesario para entender qué se observó, qué se decidió, qué se construyó y qué debería validarse después.

## Por qué importa

Este proyecto puede ser pausado y retomado después.

Si la documentación no preserva la intención, el trabajo futuro podría partir desde la solución técnica y no desde el problema real que la justificaba.

También importa porque esta estructura sirve como caso real para validar VSlices Method y VSlices Docs Standard.

## Siguiente acción

- Mantener documentos cortos y orientados a intención.
- Documentar iteraciones según el alcance real del trabajo.
- Evitar agregar documentos que no ayuden al yo futuro a retomar el contexto.
- Usar esta documentación como Project Story para VSlices.
