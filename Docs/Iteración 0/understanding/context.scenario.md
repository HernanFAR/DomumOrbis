# Contexto del escenario - It. 0

## Nota

La compra mensual de despensa aparece como una necesidad relevante porque conecta dinero disponible, planificación doméstica, carga ejecutiva, tiempo limitado y continuidad del hogar.

La situación observada es que, si la lista mensual de compra no está preparada antes del pago, la compra puede retrasarse o no realizarse a tiempo. Cuando eso ocurre, el dinero destinado a despensa puede dispersarse antes de cubrir necesidades esenciales.

## Contexto

Hoy, la lista de compra debe reconstruirse manualmente en la aplicación de Líder al inicio de cada mes. Aunque la compra suele ser bastante estable, el proceso depende de que exista tiempo, energía y disponibilidad mental para volver a preparar la lista antes de comprar.

Esto se vuelve frágil cuando hay alta carga de trabajo, cansancio o situaciones personales que reducen la capacidad de planificación. En particular, el TDAH aumenta el riesgo de que una tarea importante, pero repetitiva, no se ejecute en el momento correcto.

Hay aplicaciones de mercados que pueden mostrar compras anteriores, pero eso no resuelve completamente la necesidad observada, porque no permite transformar fácilmente una lista mensual estable en una compra preparada o reutilizable.

## Por qué esto importa

Este contexto importa porque permite identificar una necesidad real, urgente  (reducir dispersión del dinero) que se puede solucionar mediante Slice-First.

El sistema futuro podría llegar a preparar carritos, conectarse con un mercado, marcar listas como pagadas o incluso pagar una lista bajo confirmación humana. Sin embargo, el problema actual no es todavía pagar automáticamente.

El problema actual es que la intención de compra mensual debe existir antes del momento crítico del pago.
